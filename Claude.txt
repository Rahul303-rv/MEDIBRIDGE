Structure of every phase:

The discipline this enforces:

One phase per Claude session — fresh context each time, three docs always attached
No phase advances until the manual script passes AND automated tests pass AND the exit checklist is fully ticked
Commit and git tag phase-N-complete at every gate, so you can roll back if something downstream surfaces an issue
A TODO.md for anything you notice mid-phase that doesn't belong in the current phase — keeps scope from leaking
